# SHARKS SCORE APP スキル記録

## 概要
Next.js、TypeScript、Google Sheets APIを使用して構築された、少年サッカー向けのリアルタイム試合記録・速報システムです。

## 主要機能と実装詳細

### 1. 試合記録管理 (Match Management)
- **入力モード**: 「簡易（スコアのみ）」と「通常（すべての詳細）」を切り替え可能。
- **継続入力ワークフロー**: 試合保存後に、同じ対戦相手・会場で続けて次の試合を記録（TRM等で便利）。
- **試合形式の動的切替**: 「前後半（ハーフ）」と「1本（一本勝負）」に対応。1本モードではフェーズ管理が簡略化され、「進行中」と表示される。
- **自動集計**: 前後半のスコアを編集すると自動的に合計スコアが更新される。

### 2. ライブ配信 (Live Recording)
- **リアルタイム反映**: ライブフラグをオンにすることで、トップページの試合一覧に「LIVE」バッジが表示される。
- **フェーズ管理 (Match Phase)**: 
  - 前後半モード: 試合前 -> 前半進行中 -> HT -> 後半進行中 -> 終了
  - 1本モード: 試合前 -> 進行中 -> 終了
- **途中保存機能**: 入力画面を閉じずに、現在のスコアをGoogle Sheetsに反映（速報用）。
- **状況連動スコア加算**: 「前半進行中」に選手ボタン等でスコアを入れると、前半スコアと合計スコアが同時に加算される。

### 3. 多人数編集・排他制御
- **楽観的ロック**: データの更新日時（lastUpdated）を比較し、古いデータでの上書きを防止。
- **編集ロック (Exclusive Lock)**: `api/matches/lock` エンドポイントを介して、1分間の期限付きロックを管理。
  - 他のユーザーが編集中の場合は警告が表示され、編集画面に入れない。
  - 画面を離れるか保存するとロックが解除される。

### 4. マスタデータ管理 (Spreadsheet Sync)
- **会場・対戦相手・選手マスタ**: 入力時にサジェスト（Autocomplete）を表示。新しい項目は自動的に「CommonMasters」シートに追加・更新。
- **使用回数カウンタ**: 頻繁に使う項目が上位に来るよう、使用回数を管理。

### 5. UI/UX・ブランディング
- **レスポンシブデザイン**: スマートフォンでの試合現場入力を想定した、大きく押しやすいボタン配置。
- **カスタムブランディング**:
  - タイトル: SHARKS SCORE APP
  - アイコン: サメとサッカーボールのカスタムロゴ。
  - マニュアルページ: 運営者向けの使い方解説を完備。

## 技術スタック
- **Framework**: Next.js (App Router)
- **Language**: TypeScript
- **Auth**: NextAuth.js (Google Auth等)
- **Database**: Google Sheets (google-spreadsheet library)
- **Design**: Tailwind CSS (Lucide-like UI design)

## 注意事項・継続課題
- Google Sheets APIのレートリミットに配慮し、不要なリロードを避ける。
- 選手名の重複管理や、年度ごとのマスタ切り替え。
- ライブ配信のラグを最小限にするためのキャッシュ戦略。
